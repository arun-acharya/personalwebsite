Responsive Design

What is Responsive Design?


Responsive Design is process of designing websites that is readable and enjoyable on all type of devices regardless of their height and width. A responsive web design will automatically adjust for different screen sizes and viewports.

It keeps images from being larger than the screen width, and prevents visitors on mobile devices from needing to do extra work to read your content. Responsive Design use various css media queries and flexible grid. Bootstrap is one of the popular framework for mobile first responsive design which help web designers to create mobile first responsive web design without any hardship.


Why Responsive Design?


Internet is daily used on various types of devices having different height, screen resolution. So, in order to look yor website attractive and readable on different type of devices responsive design is used.In the past, developers built more than one site in order to accommodate different screen sizes. But we don't have to do such thing today because of responsive designing.

Responsive design will make your site mobile-friendly, improve the way it looks on devices with both large and small screens, and increase the amount of time that visitors spend on your site. The ultimate goal of responsive design is to avoid the unnecessary resizing, scrolling, zooming, or panning that occurs with sites that have not been optimized for different devices. It is often very difficult to navigate these sites, and it may even cost you potential customers who become frustrated with trying to figure out how to do something.


Responsive Images


What is responsive images?

Responsive image is a method of providing the browser with multiple images sources so that it can be replaced depending on display density, size of image element, or any other factor.


Why use responsive images?

Since internet is used on variety of screen sizes. So, having responsive images is important because it helps us deliver optimal file size, the right image for the right screen size, improves user experience and improves loading time. Having various versions of your image is important for creating a responsive image.


How to use responsive images?


There are two major ways to use responsive images:
  1. Using CSS

    2. By using css you can make image responsive for the most part. First, add image element and give it a class in HTML. After adding image element refer it on css by its class and set the CSS width property to 100% and height to auto:

    Example
    HTML
    <img src="nature.jpg" alt="Nature" class="responsive">
    CSS
    .responsive {
      width: 100%;
      height: auto;
    }

  2. Using Bootstrap

    3. It is far simpler to make images responsive on bootstrap. You just have to add image element on HTML and add bootstrap class of "img-fluid" and its done.

    Example
    HTML
    <img src="nature.jpg" alt="Nature" class="img-fluid">

Responsive Topography


What is responsive topography?

Responsive design is the practice of writing CSS styles in a way that displays the website correctly on all type of devices. Responsive topography generally deals with written content of a website and makes them responsive and readable on all devices types.

Using media queries to target different breakpoint widths and writing different sets of CSS at different breakpoints are at the core of responsive topography design.


Why need responsive topography?

Responsive web design, changes the way we need to look at our web typography. When designing for a multitude of screen sizes, we look to designing system and find ways to maintain consistencies across all the various ways our layout may be rendered.

Strong typography helps keep the consistent feel across all these context When your content appears with consistent typography across all the devices your audience sees it in, your site feels more familiar, even when the exact layout details may have changed.


How to use responsive topography?

  1. To use responsive topography we first need to put the following meta tag in head of your .html file:


    <meta name="viewport" content="width=device-width, initial-scale=1.0">

  2. After that we need to use media queries in css to make the layout or text responsive. It make be quite time consuming but the final result will be fruitful. A small example is given below:


    /* Use a media query to add a breakpoint at 800px: */
    @media screen and (max-width: 800px) {
      h1 {
        font-size: 6rem; /* The font-size is 6rem, when the viewport is 800px or smaller */
          }
    }

    /* Use a media query to add a breakpoint at 400px: */
    @media screen and (max-width: 400px) {
      h1 {
        font-size: 3rem; /* The font-size is 3rem, when the viewport is 400px or smaller */
          }
    }


Tips for Responsive Design


Now that you have understand responsive design, following are some helpful tips to make responsive design easier:

  1. Use fluid grid layout

    2. A grid sizes the elements of your site proportionally, rather than making them one specific size. This makes it easy when sizing things for different screens: the elements will respond to the size of the screen (that is, the grid), not the size they’re set to be in pixels. A responsive grid is often divided into columns, and the heights and widths are scaled. Nothing has a fixed width or height. Instead, the proportions depend on how large the screen is.


  2. Use pre-designed layout

    3. If you do not have the time or the desire to design a responsive website yourself, you can “cheat” by using a theme or pre-designed layout that does the work for you. This means that all you will have to worry about doing is updating the colors, branding, and content to match your company’s needs. If you use WordPress, there are plenty of both free and paid themes available that are responsive right out of the box. The same applies for many leading ecommerce providers who offer themes on their site.


  3. Use rem instead of px

    4. REM stands for root em. Basically, it provides a way to specify lengths as fractions of the root element's font size. I recommend using rem units for fonts, if only because it makes it easier for you, the developer, to change sizes. It's true that users very rarely change the default font size in their browsers, and that modern browser zoom will scale up px units. Rem is better choice than px because it also help when using media queries in css.


  4. Start With a Mobile First Approach:

    5. You should build the mobile site first, and then scale up and build the tablet and desktop designs of the website. One of the major concerns for all three (mobile, tablet and desktop) is the logo and/or text. If the text is easy to read on the mobile device, then you should not have any issues with the desktop or tablet.


  5. Build the Layout of Your Site First:

    6. I recommend you to build the entire layout of the interface before doing coding. In this way, you will have the website-look that your client wants. While building a mobile site layout, one important consideration is to make the buttons large enough for a fingertip. Another consideration is to keep the website design simple and functional. I’ve seen many designers to add too much to a mobile interface, but it is not required as it can cause many design and usability problems.


  6. Use bootstrap

    7. Bootstrap is the web’s most popular framework for designing responsive web applications. It gives front end developers the ability to quickly build fluid web pages with a set of common HTML and CSS components. Its easy to use because it doesn't require learning a different language just a set of naming conventions applied to HTML class names.
